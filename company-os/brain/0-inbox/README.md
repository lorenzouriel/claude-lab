# 0-inbox/ — drop zone

Drop here any file you want CompanyOS to read once: a Google Ads export
CSV, a sales spreadsheet, a contract PDF, a meeting transcript, a report
screenshot.

Skills like `/analyze-data` and `/ads-report` read directly from this
folder. You drag the file in, call the command, and get the summary back.

This is **input, not a final artifact**. What matters from what lands here
gets routed out: into `memory/` (lasting context), `brain/1-projects/` or
`brain/2-areas/` (work), or `output/` (a generated deliverable). The inbox
stays empty between tasks.

Everything here except this README is gitignored.

When you have storage MCPs connected (Google Drive, Notion), you can ask
Claude to fetch files straight from the source — no need to download here.
