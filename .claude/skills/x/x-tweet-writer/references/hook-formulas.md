# X Hook Formulas

Ten short-form hook skeletons validated on X in 2025-2026.

---

## T1 — Contrarian Opener

**Pattern:** `Everyone [does/says/believes] X. [Pushback]. [Why].`

**Rules:**
- State the consensus first (makes you agree-able for 1 second)
- Hard pivot in line 2
- Short third line with the real claim — don't over-explain

**Skeleton:**
```
Everyone [consensus belief].

[The opposite / the nuance].

[Reason in one sentence.]
```

**Worked example:**
```
Everyone says you need 10k followers to get clients.

You don't.

3 targeted DMs to the right people beat 1M impressions every time.
```

## T2 — Number Shock

**Pattern:** `[Specific number] [surprising fact]. [Implication].`

**Rules:**
- Number must be oddly specific (9.4x, $14,320, 73 days — not round)
- Put the number first, before any context
- One-line implication, no explanation yet

**Skeleton:**
```
[Specific odd number] [surprising thing].

[What it means / why it matters].
```

**Worked example:**
```
$847/month. That's what running a full AI content stack costs me.

Here's every line item:
```

## T3 — Confession / Mistake

**Pattern:** `[N time ago] I [embarrassing/wrong action]. [What I learned].`

**Rules:**
- Be specific about the mistake — vague confessions read as humble-bragging
- The lesson must be actionable, not "I learned to be better"
- Works best when the mistake is one the reader has also made

**Skeleton:**
```
[Timeframe] ago I [specific mistake].

[What broke / what I lost].

[The actual lesson in one sentence.]
```

**Worked example:**
```
2 years ago I launched a product with zero audience.

Made $0 in month 1. Almost quit.

The mistake: I built before I wrote. Never again.
```

## T4 — Hot Take + Thread

**Pattern:** `[Bold claim] (🧵)` or `[Bold claim]. Thread:`

**Rules:**
- The claim must be falsifiable — weak hot takes die
- No hedge words: "might", "perhaps", "kind of"
- Thread marker at the end signals more is coming

**Skeleton:**
```
[Strong opinion that will make some people nod and some push back].

Thread:
```

**Worked example:**
```
Data engineers are the most undervalued role in tech right now.

Thread:
```

## T5 — Pattern Interrupt

**Pattern:** `Stop [common advice]. [What to do instead].`

**Rules:**
- The "stop doing X" must be something people actually do
- Don't make X a strawman — real, widely-practiced behavior
- "Instead" must be concrete, not just "do better"

**Skeleton:**
```
Stop [widely practiced behavior].

[Specific alternative action] works 10x better.

Here's why:
```

**Worked example:**
```
Stop writing threads with 20 tweets.

5-tweet threads with one punchy idea per tweet get 3x the engagement.

Here's the structure I use:
```

## T6 — Comparison

**Pattern:** `[X] > [Y]. Here's why:` or `[X] vs [Y]. I tested both.`

**Rules:**
- Both sides must be real options people debate
- Don't stack the deck — acknowledge where Y wins before explaining why X is better
- Works well for tools, approaches, or mindsets

**Skeleton:**
```
[Option A] > [Option B].

I [tested/used/compared] both for [timeframe].

[Result in one sentence.]
```

**Worked example:**
```
dbt Core > dbt Cloud for solo teams.

Ran both for 6 months. Cloud added $400/mo for features I never used.

Here's what Core can do that most people miss:
```

## T7 — Question Hook

**Pattern:** `Why do [people] still [do thing that seems obviously wrong]?`

**Rules:**
- Question must be genuine — not rhetorical bait with an obvious answer
- Works best when the answer is counter-intuitive
- Don't answer in the same tweet — let it sit or thread below

**Skeleton:**
```
Why do [smart/experienced people] still [behavior that seems suboptimal]?

[One line that hints at the answer without giving it.]
```

**Worked example:**
```
Why do senior engineers still write SQL in production pipelines by hand?

Because every "better" abstraction eventually becomes the thing you're abstracting away from.
```

---

## T8 — The Reveal

**Pattern:** `The real reason [X happens] is [unexpected Y].`

**Rules:**
- Reveal must be genuinely non-obvious
- Don't tell them what they already know
- Pair with a thread if the explanation needs space

**Skeleton:**
```
The real reason [common phenomenon] is [surprising root cause].

[One line that reframes how they should think about it.]
```

**Worked example:**
```
The real reason most data projects fail isn't bad data.

It's that nobody agreed on what "done" looked like before they started.
```

---

## T9 — Thread Bait

**Pattern:** `I spent [N time] [doing research/studying/building]. Here's what I found 🧵`

**Rules:**
- Time investment signals credibility — be specific
- "Here's what I found" is the classic but still works; "Here's what surprised me" adds twist
- The 🧵 emoji is the only emoji allowed on thread openers

**Skeleton:**
```
I spent [specific time] [researching/building/studying] [topic].

Here's what I found 🧵
```

**Worked example:**
```
I spent 3 months testing every AI writing tool on the market.

Here's what I found 🧵
```

---

## T10 — How-I

**Pattern:** `How I [achieved result] in [timeframe]:`

**Rules:**
- Result must be verifiable or at least plausible
- Don't inflate — "grew to 1M followers in 7 days" reads as fake
- Colon at the end signals thread below

**Skeleton:**
```
How I [specific result] in [timeframe]:
```

**Worked example:**
```
How I went from 0 to 4k followers in 90 days without posting every day:
```

---

## Choosing the right formula

| Situation | Best formula |
|---|---|
| You have data / numbers | T2 Number Shock |
| You have a strong opinion | T4 Hot Take or T1 Contrarian |
| You made a real mistake | T3 Confession |
| You tested two options | T6 Comparison |
| You did deep research | T9 Thread Bait |
| You achieved a result | T10 How-I |
| Something counter-intuitive to say | T8 Reveal |
| Common practice that fails | T5 Pattern Interrupt |
| Genuine open question | T7 Question |
