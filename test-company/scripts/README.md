# scripts/ - CompanyOS utilities

Node.js and Python scripts that skills call when they need to do things
outside pure AI's reach (generate an image, post to social media, render
HTML to PNG).

The folder starts **empty** - each skill that needs a script has
instructions for creating it (and it is usually a one-time setup per
integration you activate).

## Common scripts

As you activate skills, this will be populated. List of what each skill
expects to find:

| Skill | Expected script | What it does |
|---|---|---|
| `/carousel` (with AI photo) | `generate-image.js` | Generates a realistic photo via OpenAI API (DALL-E 3) |
| `/carousel` (PNG render) | `render.js` (generated by carousel, stays in the content folder) | Playwright takes a 1080x1350 screenshot of each slide |
| `/approve-post` | `post-instagram.js` | Publishes the carousel to Instagram via Meta Graph API |
| `/approve-post` | `post-facebook.js` | Publishes the carousel to Facebook via Meta Graph API |
| `/google-ad` | (none - generates CSV directly) | - |
| `/ads-report` | (reads CSV exported from the platforms) | - |

## Common prerequisites

Most scripts depend on:

**Node.js 20+** installed on the machine

**.env** at the project root with API keys:
```bash
OPENAI_API_KEY=sk-...               # for generate-image.js
META_PAGE_ACCESS_TOKEN=...          # for post-instagram.js + post-facebook.js
META_PAGE_ID=...
META_IG_USER_ID=...
SITE_URL=https://yourdomain.com
```

**Playwright** (to render HTML to PNG):
```bash
npm install playwright
npx playwright install chromium
```

## How CompanyOS handles this

When you run a skill that needs a missing script, Claude will:

1. Detect that the script is missing
2. Ask if you want to configure it now
3. Guide you through API key setup (Meta, OpenAI, etc.)
4. Create the configured script
5. Run the skill

You do not need to memorize anything. Run the skill and follow the flow.
